Cloud Network Design Choices

SplineTM


The Arista Spline architecture directly connects hosts to a pair of switches configured in an MLAG, allowing it to scale to over 2000 hosts without blocking. This extensive single-tier setup supports densely deployed applications such as IP and Ethernet storage, next-generation file systems, distributed computing, and virtualization.

               

Layer2 / MLAG



The L2LS network design is a two-tier architecture consisting of two spine switches and one or more Top-of-Rack (ToR) leaf switches. Leaf switches connect to storage, compute, service, and data center edge network elements, and can be deployed individually or in pairs for high availability. Spine switches aggregate the leaf switches and provide a high-speed backbone./p>

               

Layer3 / ECMP



The L3LS network design is a two-tier architecture consisting of 2 to 128 spine switches and one or more Top-of-Rack (ToR) leaf switches. Leaf switches connect to storage, compute, service, and data center edge network elements and can be deployed individually or in pairs for high availability. Spine switches aggregate the leaf switches and provide a high-speed backbone.

               

L2 over Layer 3 VXLAN

Building on the same two-tier architecture of the L3LS design, the Arista Networks L3LS-V Solution incorporates Virtual eXtensible Local Area Network (VXLAN). VXLAN is an open IETF specification that standardizes an overlay encapsulation protocol, enabling the relay of Layer 2 traffic over IP networks.
